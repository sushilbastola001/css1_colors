HWB

HWB stands for Hue, Whiteness, and Black. Hue value is in either degree, radian, or gradient. If only number is specified then it is assumed as degree value. Whereas value of W and B are in percent. If W=0% and B=100% than it produces black. W=100% and B=0% produces white. And W=505 and B=50% produces gray.

Example with: W=50% and B=50%

LCH

LCH stands for Lab, Chroma, and Hue. LCH can be used to create palettes across entirely different colors, with predictable results. Please note that LCH hue is not the same as HSL hue and LCH chroma is not the same as HSL saturation. L specifies CIE lightness: 0 % black and 100% white. For C, minimum value is 0 and practically, maximum value is 230. The third argument H is the hue angle. 0deg points along the positive "a" axis (toward purplish red), 90deg points along the positive "b" axis (toward mustard yellow), 180deg points along the negative "a" axis (toward greenish cyan), and 270deg points along the negative "b" axis (toward sky blue).

Example with: L=10%, C=100 and H=10deg

Clamp

clamp() enables selecting a middle value within a range of values between a defined minimum and maximum. It takes three parameters: a minimum value, a preferred value, and a maximum allowed value. The minimum value is the smallest (most negative) value. If the preferred value is less than this value, the minimum value will be used. The preferred value is the expression whose value will be used as long as the result is between the minimum and maximum values. The maximum value is the largest (most positive) expression value. The value of the property will be assigned if the preferred value is greater than this upper bound.

Example of Clamp with minimum, preferred, and maximum value.
